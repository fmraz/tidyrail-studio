# Launch Plan

## Current Launch Strategy

Tidyrail Studio will launch with free products first. No payment processing, subscriptions, locked Pro features, or artificial limits are part of the first public release.

## Phase 1: Private Release Candidate

- Finish Renewal Desk.
- Create company website.
- Prepare release zip.
- Prepare docs, privacy, terms, roadmap, FAQ, and support pages.
- Buy domain after founder payment approval.
- Configure hosting.

## Phase 2: Public Soft Launch

- Publish Tidyrail Studio website.
- Publish Renewal Desk as a free download and free web app.
- Serve the website through GitHub Pages at `tidyrailstudio.com`.
- Share only in appropriate places with honest copy.
- Collect feedback.

## Phase 3: Distribution Expansion

- Prepare GitHub release if public repository is approved.
- Prepare Product Hunt launch.
- Prepare desktop packaging.
- Prepare mobile structure only after web usage validates the product.

## Phase 4: Future Monetization Review

Review monetization only after:

- Users have tried the free product.
- Feedback identifies real value.
- Support burden is understood.
- Platform distribution is stable.

## Manual Approval Required

- Future domain purchases.
- Hosting purchase if any.
- Account registration where personal verification is required.
- DNS changes in VEDOS if browser access or 2FA is required.
- Store submissions.
- Social posts.
- Email sending.
- Developer memberships.
