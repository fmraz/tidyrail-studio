# Support Workflow

## Channels

Initial support should use the website contact/support pages. Do not publish a support email address or ticketing system until the workflow is approved.

## Response Priorities

1. Data loss or export/import issue.
2. Download or launch issue.
3. Account or privacy issue after auth is enabled.
4. Usability issue.
5. Feature request.

## Templates

Use concise, honest replies. Never promise timelines unless a release is already scheduled.

## Privacy

Do not ask users to send sensitive files publicly. If diagnostic files are needed later, create a private support process first.
