# Maintenance Plan

## Weekly Studio Review

- Review unfinished products.
- Check release-candidate blockers.
- Update roadmaps and changelogs.
- Review support questions or likely FAQ gaps.
- Add only high-quality product ideas after progress has been made.

## Patch Strategy

- Critical data-loss bugs: patch immediately before new features.
- Broken release candidates: highest priority.
- UX confusion: fix before marketing expansion.
- Feature requests: score against product focus and maintenance burden.

## Versioning

- `0.x`: MVP and private testing.
- `1.0`: first free public release candidate.
- Patch releases: bug fixes and small UX improvements.
- Minor releases: contained features that do not change the product's core promise.
