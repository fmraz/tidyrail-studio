Security posture.

Tidyrail Studio keeps the current release intentionally small: static pages, local-first product data, no advertising scripts, and no active cloud account system until the backend is approved and tested.

Current website baseline

The public site is built as static HTML, CSS, and JavaScript. Browser-side protections are included in every HTML page, while HTTPS enforcement depends on GitHub Pages finishing a valid certificate for the custom domain.

Security controls

  • Content Security PolicyPages restrict scripts, forms, images, workers, and connection targets to the site and approved future Supabase endpoints.
  • Local-first storageRenewal Desk records are stored in this browser by default and can be exported by the user.
  • No service keys in frontend codeFuture cloud sync is gated and must not expose privileged Supabase service role keys in public assets.
  • No ad or tracking pixelsThe public release does not include advertising networks or invasive analytics scripts.

Report a vulnerability

Please avoid posting sensitive exploit details publicly. For non-sensitive product issues, use the public repository issue tracker. A dedicated security email will be published only after mail handling for the domain is ready.

HTTPS status

The DNS records point to GitHub Pages. HTTPS enforcement should be enabled in GitHub Pages only after the certificate presented for tidyrailstudio.com is valid for that hostname.
