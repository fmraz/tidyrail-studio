# Patch Plan

## Release Loop

1. Identify bugs and regressions.
2. Prioritize by data loss, broken core flow, security, install failure, then UI polish.
3. Reproduce the issue.
4. Fix the smallest affected area.
5. Test affected flows.
6. Update changelog and release notes.
7. Rebuild packages.
8. Update website download links.
9. Verify public links.

## Patch Priority

- P0: data loss, security issue, broken download, broken app launch
- P1: broken core workflow
- P2: serious UX/accessibility issue
- P3: polish and copy fixes
